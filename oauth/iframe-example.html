InfinityElements are embedded using srcdoc (opaque origin) for security. For popup-based OAuth to work, you must include these sandbox permissions. The element runs in a fully isolated environment and does NOT communicate with the parent page.

üîí Required Sandbox Permissions

  • allow-scripts - Enables JavaScript execution
  • allow-popups - Allows opening popup windows
  • allow-forms - Enables OAuth form submissions and redirects

Note: The popup will inherit the sandbox restrictions, but allow-forms enables OAuth redirects to work properly. The popup communicates back to the iframe via postMessage (same-origin).

üåê Required Allow Permissions

  • local-network-access - Allows network requests to local/private network resources

Note: The allow attribute controls feature policies and is separate from the sandbox attribute. InfinityElements are embedded with srcdoc, giving the iframe an opaque origin (null origin) for maximum security.

üìù Iframe Code

Production configuration using srcdoc (opaque origin):

<!-- OAuth SSO Demo InfinityElements -->
<iframe
  sandbox="allow-scripts allow-popups allow-forms allow-popups-to-escape-sandbox"
  allow="local-network-access"
  srcdoc="<!-- element HTML content -->"
></iframe>

Note: This demo page uses src="./index.html" for demonstration purposes only. In production, components are embedded via srcdoc.
